# UI 设计语言（DESIGN）

> 版本：v4.0 · 2026-07-10
> 状态：已按 CMC/Coinbase 风格完成 P0+P1 重构
> 产品方向：国内期货与 A 股市场发现 + 自选 + 基础模拟盘 + 事件资讯 + 本地数据 + 引用式 AI
> 参考气质：Coinbase / CoinMarketCap 的清晰信息层级、低噪声金融 UI + 桌面研究终端的信息密度
> 组件体系：shadcn/ui New York + Tailwind + `frontend/src/design/tokens.css`

> 说明：CMC/Coinbase 风格参考的是其产品结构与信息密度，不是业务照搬。复杂交易入口（止损/止盈/OCO/移动止损/条件单）和历史行情回放训练已收敛为设置页实验区功能，不作为主业务入口。

---

## 1. 设计目标

ThisIsMyQuant 的界面应像一个研究员和交易练习者每天打开的本地市场工作台，而不是营销首页或泛资讯门户。设计目标：

1. **快速判断**：首屏能看出五大板块强弱、关注列表异动、最新决策流和待处理报告。
2. **练习交易**：模拟账户、下单面板、持仓、委托、成交和风险度必须随时可见。
3. **解释行情**：价格变化必须能向事件资讯、因子、外盘联动或模拟持仓继续下钻。
4. **可信边界**：实时、延迟、历史、估算、错误数据和模拟交易都要明确标识。
5. **低噪声**：避免大面积装饰、营销式 hero、过度渐变和嵌套卡片。
6. **桌面优先**：适合长时间盯盘、研究和阅读，信息紧凑但不拥挤。

最新 UI 高保真设计稿：`docs/CMC_PRODUCT_VISUAL_DESIGN.html`。该文件为本地设计产物，已加入 `.gitignore`，不作为仓库提交物。

## 2. 页面矩阵

| 页面 | 目标 | 关键组件 |
|---|---|---|
| 总览 | 市场摘要 + 热门发现 | 账户摘要、板块概览、决策流、报告任务 |
| 市场 | 期货/A 股统一发现 | AssetTable、MarketLeaderboard、MarketFilters、MiniSparkline |
| 标的详情 | 统一资产详情 | AssetHeader、KlinePanel、AssetStatsGrid、AssetDetailTabs |
| 自选 | 关注列表与异动 | WatchlistTable、WatchlistGroupTabs、WatchlistEventPanel |
| 模拟盘 | 基础虚拟交易 | OrderTicket、PositionTable、OrderTable、TradeTable、CashFlowTable |
| 事件资讯 | 事件流聚合 | EventTimeline、EventFilterBar、EventImpactTags |
| 数据库 | 数据资产中心 | DataDomainGrid、DataDomainTable、DataDomainActions |
| AI | 引用式分析 | AiTaskPanel、AiReportCard、AiSourceList |
| 报告 | 报告归档 | 报告列表、详情、筛选 |
| 状态 | 系统健康 | 数据源、任务、LLM、数据库状态 |
| 设置 | 本地配置 | Provider、Key、关注列表、调度、偏好、实验区 |

旧页面按以下方式收敛：

- 行情、品种 → **市场** / **标的详情**。
- 资讯、日历、异动 → **事件资讯**。
- 因子、助手、Copilot → **AI**。
- 复盘、回放，以及复杂订单（止损/止盈/OCO/移动止损/条件单）→ **设置页实验区**。

## 3. 信息架构

主导航改为固定顶部导航，顺序与用户工作流一致：

```
总览 → 市场 → 自选 → 模拟盘 → 事件资讯 → 数据库 → AI → 设置
```

页面布局遵循三层结构：

1. **页头**：页面标题、当前板块/品种、主要操作、更新时间。
2. **主内容**：图表、列表、时间线、表格、因子矩阵等核心工作区。
3. **辅助信息**：数据质量、来源、错误态、任务状态、关联入口。

不要在页面顶部放营销文案、产品介绍或大面积 hero。用户进入应用后应直接看到可操作的分析界面。

## 4. 视觉原则

| 原则 | 说明 |
|---|---|
| 浅色优先 | 默认浅色 Coinbase/CMC 风格，深色主题为可选项，降低日常阅读和浏览疲劳。 |
| 细线分层 | 面板使用 1px 边框和轻微背景差，不使用重投影。 |
| 金融中性 | 主色用于操作和焦点，涨跌色只用于行情语义。 |
| 信息密度适中 | 总览可密集，详情页要给图表和阅读留足空间。 |
| 不嵌套卡片 | 页面 section 不做浮动卡片，卡片只用于重复项、面板或模态。 |
| 可追溯 | 资讯、因子、报告必须保留来源/时间/质量标识。 |

## 5. Token 与颜色

项目 token 位于 `frontend/src/design/tokens.css`，Tailwind 映射位于 `frontend/tailwind.config.ts`。

| 用途 | 规则 |
|---|---|
| 背景 | 默认浅色主题页面底色接近白色/浅灰，深色主题可选。 |
| 主操作 | 使用单一 primary 色，不为不同模块发明新的 CTA 色。 |
| 上涨 | 仅用于价格、涨幅、正向行情信号。 |
| 下跌 | 仅用于价格、跌幅、风险/负向行情信号。 |
| 警告 | 用于数据陈旧、任务失败、重要日历事件。 |
| 数据质量 | 使用 badge + 文案，颜色必须克制。 |
| 模拟交易 | 所有下单、持仓、成交和资金组件必须显示“模拟”标识。 |
| A 股市场 | 指数、行业、个股、财报、筛选器需显示数据源、报告期和复权口径。 |

避免单一紫蓝、单一咖啡/棕色、纯渐变背景等一眼主题化的界面。

## 6. 字体与数字

| 内容 | 建议 |
|---|---|
| 页面标题 | 20-24px，600，短标题。 |
| 面板标题 | 14-16px，500/600。 |
| 正文 | 13-14px，适合阅读。 |
| 辅助信息 | 11-12px，低对比。 |
| 价格/合约/时间 | 等宽字体。 |

不要用视口宽度动态缩放字号；不要使用负字距。长中文标题需要换行或截断，不能挤出按钮和卡片。

## 7. 组件规则

| 组件 | 规则 |
|---|---|
| Button | 明确命令才用文字按钮；工具类操作优先图标按钮并提供 tooltip。 |
| Badge | 用于板块、品种、触发类型、数据质量和来源。 |
| Tabs | 用于同页面视图切换，如板块/维度/触发类型。 |
| Table/List | 行情、资讯、报告、日历优先使用可扫描列表。 |
| Chart | 图表应占据主要视觉空间，不放进装饰性容器。 |
| Timeline | 决策流和报告流式输出使用时间线，但节点信息要短。 |
| Empty/Error | 空态说明原因和下一步操作，不使用营销插画。 |
| Order Ticket | 仅保留市价/限价、买卖、开平、手数、费用估算；复杂订单移入设置页实验区。 |
| Position Row | 持仓行固定展示方向、手数、均价、浮盈、保证金、风险操作。 |
| Stock Screener | 筛选器必须显示条件、命中数、快照时间、使用的数据报告期。 |
| Financial Table | 财务表要突出报告期、同比、环比、异常值和数据源。 |

卡片圆角保持 6-8px，除非已有组件要求更大。不要把卡片套进卡片。

## 8. 数据质量展示

所有核心数据组件都要显示更新时间或质量状态。

| 状态 | UI 表达 |
|---|---|
| `live` | 正常低调 badge，可省略警告色。 |
| `history` | 标注历史数据。 |
| `reference` | 标注参考来源，如外盘参考。 |
| `estimated` | 明确写“估算”，不得只显示数值。 |
| `pending` | 显示待更新或待接入。 |
| `stale` | 显示旧数据时间，使用警告语义。 |
| `error` | 显示错误、来源和重试入口。 |

报告中若使用估算或参考数据，应在对应段落说明。

## 9. 五大板块 UI 表达

板块颜色只用于辅助识别，不作为大面积背景。

| 板块 | 建议表达 |
|---|---|
| 黑色建材 | 中性冷灰，突出库存/需求/政策。 |
| 有色贵金属 | 金属蓝灰，突出美元利率/库存/避险。 |
| 农产品软商品 | 低饱和绿色，突出天气/季节/进口。 |
| 能源化工 | 低饱和青色或琥珀，突出原油/装置/库存。 |
| 航运运价 | 深青或海蓝点缀，突出地缘/舱位/需求。 |

金融期货不作为 v1 导航或板块入口。

## 10. 关键页面设计要求

### 总览

- 首屏展示模拟账户权益、今日盈亏、风险度、板块概览、风险提示、决策流和报告任务。
- “下一步要看什么”应由数据驱动，而不是静态说明文字。
- 决策流节点必须标明来源类型：行情、资讯、日历、异动、报告。

### 市场

- 作为期货与 A 股的统一发现入口，不再区分“行情”与“A 股”独立页面。
- 默认展示市场列表、领涨/领跌、板块过滤和搜索，支持按资产类型（期货/A 股/ETF）筛选。
- 列表行包含名称、最新价、涨跌幅、成交额/持仓量、MiniSparkline 与数据质量标识。

### 标的详情

- 统一展示期货品种与 A 股个股的资产头部、K 线、关键统计和详情 Tab。
- K 线支持周期切换与技术指标；A 股需显示复权口径与数据源。
- 提供“加入自选”与“触发分析”主操作。

### 自选

- 关注列表按分组展示，支持期货与 A 股混合关注。
- 行内高亮异动状态（价格、成交量、新闻事件），可快速跳转详情或分析。
- 自选股/品种的关联事件在独立面板聚合。

### 模拟盘

- 下单面板与持仓/委托/成交同屏，减少来回切换。
- 任何可点击下单入口都必须写明“模拟”。
- 提交前展示保证金占用、手续费和成交风险预估。
- 仅保留市价/限价、买卖、开平、手数等基础委托；复杂订单在设置页实验区。
- 拒单原因必须明确，如资金不足、超最大手数、合约规则缺失。

### 事件资讯

- 将资讯、财经日历、异动信号统一为事件时间线。
- 事件需展示来源、影响品种/板块、重要性、数据质量与操作入口。
- 异动事件可触发分析任务，生成的报告进入报告中心。

### AI

- 以引用式分析为核心，任务面板展示手动分析、每日简报、批量分析、异动快评等任务状态。
- 报告卡片必须列出引用来源（行情、资讯、日历、因子、报告片段）。
- 问答界面以研究为中心，不做聊天娱乐化；流式输出失败时保留已生成片段和重试入口。

### 报告

- 报告列表支持按品种、触发类型、日期筛选。
- 详情页保留免责声明与数据来源说明。

### 数据库

- 以数据域（行情、资讯、报告、模拟交易、复盘等）为单位展示本地资产。
- 支持导出、导入、备份和清理操作，所有批量操作需二次确认。

### 状态

- 展示 SQLite、行情、金十、日历、LLM、后台任务与模拟引擎健康状态。
- 错误状态必须可见，并提供重试或查看日志入口。

### 设置

- 配置 Provider、凭据、关注列表、调度、UI 偏好。
- **实验区** 承载历史行情回放、交易复盘、止损/止盈/OCO/移动止损/条件单等进阶功能，不作为主导航入口。

## 11. 响应式与可访问性

- 桌面优先，最小宽度下允许侧栏收缩或内容横向表格滚动。
- 所有 icon-only 按钮必须有 tooltip 或 `aria-label`。
- 颜色不能作为唯一信息来源，涨跌、质量、状态必须有文字或符号。
- 文本不得与按钮、图表、卡片内容重叠。

## 12. 工程落地

| 文件 | 职责 |
|---|---|
| `frontend/src/design/tokens.css` | CSS 变量与主题 token。 |
| `frontend/src/index.css` | 全局样式、字体和辅助类。 |
| `frontend/src/components/ui/` | shadcn/ui 原子组件。 |
| `frontend/src/components/AppShell.tsx` | 顶部导航和应用框架。 |
| `frontend/src/pages/` | 页面级产品体验。 |
| `frontend/e2e/` | UI mock E2E 对关键页面做可访问性冒烟。 |

新增 UI 时，先查是否已有 shadcn 或本地组件可复用；只有当重复复杂度明确出现时再抽象新组件。
