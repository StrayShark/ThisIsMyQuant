> 项目：**ThisIsMyQuant**
> 文档版本：v1.0 · 2026-06-24
> 状态：初稿

---

## 1. 项目背景

国内商品期货与航运运价市场缺乏一个**轻量、可自托管、对接大模型**的一体化分析工具。
现有方案要么重（vnpy 桌面端、文华财经、博易大师），要么闭源不可定制（同花顺、东财），
要么只有数据没有分析（天勤、聚宽）。

本项目的目标是：**自建一套 Tauri 桌面期货分析应用**，对接 AKShare K 线与金十资讯，
用 Lightweight Charts 呈现 K 线，并接入大模型生成走势分析报告，
帮助个人交易者完成「看盘 → 理解 → 决策」的闭环。

## 4. 非功能需求

| 类别 | 要求 |
|---|---|
| **性能** | Tick → 图表端到端延迟 < 500ms；K 线查询 1 万根 < 200ms |
| **可用性** | 数据源失败静默降级并可恢复；LLM 失败降级（重试 + 切换提供商）；CTP 断线自动重连留待 M6 |
| **安全** | 凭据加密落库；不向第三方上传用户数据；CORS 白名单 |
| **可移植** | macOS / Linux / Windows Tauri 桌面包 |
| **可扩展** | 数据源、LLM、指标均为插件式，新增不改核心 |
| **可观测** | 结构化 JSON 日志；关键链路 traceId |

## 5. 用户故事（典型场景）

### 故事 1：晨间复盘
> 作为个人交易者，我希望早上打开软件，就能看到昨夜外盘和早盘前 30 分钟的 K 线，
> 以及大模型生成的「今日关注品种走势分析」，这样我能在 9:00 开盘前快速定调。

**验收**：首页「今日报告」卡片默认展示关注列表的当日分析；K 线图默认加载主力连续合约 1h 周期。

### 故事 2：盘中实时盯盘
> 作为盯盘用户，我希望在图表上看到实时跳动的价格和成交量，
> 并且当某品种 5 分钟内涨跌幅超 1% 时，收到一条大模型的快速点评。

**验收**：WebSocket 实时推送；条件触发器生成「异动快评」，顶部通知栏 + 报告列表同时落库。

### 故事 3：品种深度研究
> 作为研究用户，我想选择「螺纹钢 rb」，查看它的多周期图、关联品种（铁矿/热卷）、
> 近 30 天的分析报告时间线，并和模型对话追问「当前基差情况」。

**验收**：品种详情页含多周期缩略图、关联品种矩阵、报告时间线（用 Cursor 的 AI 时间线柔和色板呈现）、AI 问答侧栏。

### 故事 4：自托管与隐私
> 作为自托管开发者，我希望整套软件跑在我自己的机器上，
> 数据不外传，LLM 也可以用我自己的 API Key，甚至本地 Ollama。

**验收**：Tauri 应用本地运行；`.env` 配置 LLM Key；OpenAI 兼容协议可接 Ollama。

## 6. 范围界定（In / Out of Scope）

### In Scope（v1.0）
- 国内期货近实时行情（v1 使用 AKShare/Sina 主力连续合约轮询）
- K 线图表（TradingView Lightweight Charts）
- 每日定时 + 实时大模型分析
- 本地数据持久化与回看
- Web 前端（响应式，桌面优先）
- 自托管桌面部署（Tauri 打包）

### Out of Scope（v1.0，留待后续）
- 实盘下单交易（仅分析，不碰交易，合规与风险优先）
- 复杂量化策略回测引擎（vnpy.alpha 那套，后续可接）
- 移动端原生 App
- 多用户账号体系（单租户自托管）
- 期权定价、希腊值

## 7. 验收标准（DoD）

每个功能模块完成需满足：
1. 单元测试覆盖率 ≥ 70%（核心数据/分析模块）
2. 关键接口有集成测试
3. 文档同步更新（API、使用说明）
4. `cargo test`（Rust）、`eslint` / `tsc`（前端）零 error
5. 本地 `pnpm tauri:dev` 可启动并跑通主流程
