# 国内期货与 A 股数据源分析（DATA_SOURCES）

> 版本：v1.2 · 2026-07-09  
> 当前选型：期货 AKShare/Sina 主力行情 + 金十资讯/财经日历 + Yahoo 外盘参考 + A 股 AKShare/Baostock + 本地模拟交易数据库  
> 后续选型：CTP/SimNow、TQSDK、IB、QuestDB 等作为增强项

---

## 1. 当前数据源口径

ThisIsMyQuant v1 是模拟盘与本地数据库工具，不是实盘交易终端。当前不直接对接 CTP 行情柜台、券商柜台或真实交易接口。期货默认使用主力期货数据和中文品种名展示，内部符号采用 `RB0`、`AU0`、`SC0` 等主力连续代码。A 股默认使用沪深北股票代码、指数、行业/概念和财务数据。模拟订单、成交、持仓、资金和复盘记录全部保存在本地 SQLite。

| 数据类别 | 当前来源 | 产品用途 | 数据质量口径 |
|---|---|---|---|
| 国内主力行情 | AKShare / Sina | K 线、涨跌幅、异动、技术指标 | 近实时或延迟，标记 `live/stale/error`。 |
| 历史 K 线 | AKShare | 回填、指标计算、报告上下文 | 历史数据，标记 `history`。 |
| 期货资讯 | 金十期货分类 | 决策流、资讯分类、报告上下文 | 低频抓取，标记来源和时间。 |
| 财经日历 | 金十日历/MCP 备用 | 宏观事件、报告提醒 | 按日查询，重要性过滤。 |
| 海外参考 | Yahoo Finance 等 | 原油、黄金、铜、农产品外盘联动 | 日级或延迟参考，标记 `reference`。 |
| 产业因子 | AKShare、金十、后续人工/公开源 | 库存、供需、价差、政策 | 缺失时使用 `pending/estimated`，不得伪装为实时。 |
| 模拟交易 | 本地撮合引擎 + SQLite | 虚拟账户、订单、成交、持仓、资金曲线、复盘 | 明确标记为 `simulation`，不产生真实委托。 |
| A 股行情 | AKShare / Baostock | 股票目录、指数、日 K、行业概念、市场宽度 | 标记 `live/history/stale/error`。 |
| A 股财务 | Baostock / AKShare / Tushare Pro 可选 | 财报、财务指标、估值、筛选器 | 标记报告期、来源和更新时间。 |
| A 股研究流水线 | Qlib 风格本地因子库 | 因子快照、打分、回测报告 | 本地生成，标记模型/因子版本。 |

## 3. 数据源详解

### 3.1 AKShare / Sina

- **内容**：国内期货主力连续行情、历史 K 线、部分基本面数据。
- **成本**：开源免费。
- **优点**：接入简单、覆盖广、适合本地分析工具快速迭代。
- **限制**：非 Tick 级实时，源站字段可能变化，频率不宜过高。
- **本项目定位**：v1 行情和历史回填主源。

实现要求：

1. 行情查询和历史回填在 Rust 适配器中完成。
2. 结果写入 SQLite 或 quote cache，前端只通过 Tauri command 读取。
3. UI 必须展示数据更新时间和质量状态。
4. 异动、报告、因子不能把延迟行情描述为交易级实时行情。

### 3.2 金十数据

- **内容**：期货资讯、专题卡片、财经日历、宏观事件。
- **来源页**：`https://datas.jin10.com/#/category/52003` 与 `https://rili.jin10.com/`。
- **本项目定位**：资讯决策流、宏观日历、LLM Prompt 上下文。
- **合规口径**：仅用于个人研究、本地聚合和来源标注，不商业转售，不作为唯一决策依据。

期货数据库接口示例：

| 用途 | 接口 | 关键参数/头 |
|---|---|---|
| 一级子分类 | `GET https://mp-api.jin10.com/api/home/category/sub` | `id=52003`，`x-app-id`，`x-version` |
| 分类卡片/新闻条目 | `GET https://mp-api.jin10.com/api/home/item` | `category_id`、`page`、`limit`、`platform=pc` |

财经日历接口示例：

| 用途 | 接口 | 关键参数/头 |
|---|---|---|
| 按日经济数据 | `GET {JINSHI_RILI_API_BASE}/data/getDay` | `date=YYYY-MM-DD`，`x-app-id`，`x-version` |
| MCP 备用 | `POST https://mcp.jin10.com/mcp` | `tools/call → list_calendar`，Bearer Token |

抓取约束：

1. 低频轮询，本地缓存，失败时静默降级。
2. 入库保留 `source`、`category_id`、`display_time`、`title`、`summary`、`url/raw_json`。
3. 分类先规则、后可选 LLM，保留置信度和方法。
4. 资讯进入报告 Prompt 时必须保留来源和时间。

### 3.3 Yahoo Finance / 海外参考

- **内容**：海外连续合约或 ETF/指数参考，如 `CL=F`、`GC=F`、`HG=F`、`ZS=F`。
- **用途**：外盘联动、夜盘前后参考、宏观风险提示。
- **限制**：通常为延迟或日级数据，不适合盘中精确交易。
- **本项目定位**：专业工作台 `overseas_links` 和 LLM 参考上下文。

### 3.4 CTP / SimNow

- **内容**：国内期货 Tick 行情与交易接口。
- **优点**：官方链路、实时 Tick、覆盖交易所。
- **限制**：C++ API、账户/前置机配置复杂、仅当日 Tick、需要断线重连。
- **本项目定位**：后续行情升级项，仅考虑行情接口；交易接口不在项目范围内。

### 3.5 TQSDK / IB / 付费数据

| 源 | 定位 |
|---|---|
| TQSDK | 国内期货历史与实时备份，可作为 CTP 之前的行情升级候选。 |
| IB | 海外期货实时/历史，适合后续扩展全球期货。 |
| RQData / 迅投研 XT | 专业付费数据源，适合团队或商业用户。 |
| TuShare / 东方财富 / 新浪 | 历史或校验补充，不作为唯一核心源。 |

### 3.6 A 股数据源

| 数据源 | 用途 | 定位 |
|---|---|---|
| AKShare | A 股实时行情、历史 K 线、指数、行业/概念、资金流、财务和公告等公开接口。 | P0 主源，覆盖广，适合快速产品化。 |
| Baostock | A 股历史行情、复权、财务和宏观等免费 Python API。 | P0/P1 备用源，适合历史数据和财务校验。 |
| Tushare Pro | 股票基础资料、行情、财务、指数、日历和资金面标准化数据。 | P1 专业增强源，需要用户 token。 |
| Qlib | 数据处理、因子、模型、回测和评估流水线。 | P1/P2 研究框架参考，本项目优先借鉴分层而非整体嵌入。 |
| pytdx / efinance | 通达信/东方财富相关公开行情接口。 | P2 行情备用，不承诺交易级实时。 |

A 股数据接入要求：

1. 股票代码统一存储为标准代码，例如 `600000.SH`、`000001.SZ`、`BJxxxx`。
2. 行情、财务、资金、公告必须记录来源、更新时间和报告期。
3. 财报数据不得与行情时间混淆，筛选器必须显示使用的最新报告期。
4. 实时行情只用于分析和模拟组合，不作为真实交易依据。
5. Tushare token 等凭据走本地加密配置，不写入日志。

## 4. 数据质量状态

所有行情、因子、资讯、日历和外盘数据进入 UI 前都应带质量语义。

| 状态 | 含义 | UI 表达 |
|---|---|---|
| `live` | 当前轮询成功，时间新鲜 | 正常状态。 |
| `history` | 历史回填或日线数据 | 标注历史。 |
| `reference` | 外盘或跨市场参考 | 标注参考。 |
| `estimated` | 由规则或历史数据估算 | 明确标注估算。 |
| `pending` | 数据待接入或待更新 | 显示待更新。 |
| `stale` | 最近请求失败但有旧值 | 显示旧数据时间。 |
| `error` | 无可用数据或源失败 | 显示错误和重试入口。 |

关键原则：缺失数据可以展示为空态或估算态，不能伪造为真实因子。

## 5. 数据架构

```
AKShare/Sina ─┐
Jin10 News  ──┼── adapters ── services ── SQLite / cache ── Tauri commands ── Frontend
Jin10 Rili  ──┤                  │
Yahoo       ──┘                  └── engine → LLM Prompt / reports / alerts
AKShare Stock / Baostock / Tushare ──▶ stock adapters ──▶ stock tables / factor snapshots
```

采集层负责外部接口；服务层负责轮询、缓存、去重、分类和降级；引擎层负责指标、维度、异动和 Prompt；前端只消费 Rust 输出的结构化视图。

## 7. 合规与风险提示

1. 本项目仅用于模拟交易、数据分析与信息呈现，不提供真实交易、荐股荐期或资产管理服务。
2. 抓取类数据源仅用于个人研究和本地缓存，应标注来源并遵守源站条款。
3. LLM 报告必须显示“仅供参考，不构成投资建议”。
4. 用户凭据仅存本地，加密落库，不进入日志和 Prompt。
5. 外部数据可能延迟、缺失或错误，产品必须在 UI 中暴露数据质量。
6. 模拟交易数据必须标记为模拟记录，不得与真实成交混用。
7. A 股模块不接券商实盘交易，不保存证券账户、交易密码或验证码。
