# 模拟盘后续功能路线图

> 版本：v0.2 · 2026-07-06
> 定位：补齐 ThisIsMyQuant 模拟盘从「能下单」到「像真盘一样练习」的能力缺口。

---

## 1. 目标

让 ThisIsMyQuant 的模拟盘具备接近国内期货真实交易终端的核心体验：

- 真实撮合规则（限价、止损、止盈、OCO、条件单）。
- 保证金、手续费、滑点、合约规则可配置。
- 风控规则（最大手数、单品种风险、强平线）。
- 历史行情回放训练。
- 交易复盘与绩效指标。
- 分析 → 交易 → 复盘的数据闭环。

## 3. 差距分析与优先级

将差距按 **用户价值 × 实现成本** 分为 P0（必须）、P1（重要）、P2（增强）三档。

| 差距项 | 优先级 | 理由 |
|---|---|---|
| 挂单轮询成交与 realistic fill | P0 | 没有它，限价单和条件单都无法正常工作，模拟盘失真。 |
| 止损 / 止盈 / 移动止损 / OCO | P0 | 文华/盘立方的核心能力，期货风险管理的基础工具。 |
| 条件单（价格触发） | P0 | 与止损止盈同属自动委托体系，用户练习高频场景。 |
| 历史行情回放训练 | P0 | 盘立方/NinjaTrader 的招牌能力，项目已设独立页面。 |
| 绩效分析指标 | P1 | 复盘价值高，已基于现有成交记录实现。 |
| 风控规则扩展 | P1 | 最大手数、单品种风险、强平线，已支持配置。 |
| 合约规则自定义 | P1 | 用户需要调整保证金/手续费以匹配自身策略，已支持配置。 |
| 图表交易标记 | P1 | 提升交易与分析的联动体验。 |
| 账单导入回放 | P1 | 盘立方核心竞争力，连接实盘与复盘。 |
| 画线下单 | P2 | 体验优秀，但依赖图表交互成熟度。 |
| 多账户/账户模板 | P2 | 练习不同策略或资金规模时的加分项。 |
| 盘口/Level 2 | P2 | 数据获取成本高，当前主力连续数据不足。 |
| 策略脚本/自动化 | P2 | 超出 v1「手动练习」定位，后续再评估。 |

## 4. 后续功能规划（分三阶段）

### 阶段一：让模拟盘“像真的一样成交”（P0，已完成 ✅）

目标：补齐撮合、订单类型和基础风控，使限价单、止损止盈、条件单可正常使用。

#### 4.1.1 撮合引擎增强（已完成）

- 在 `services/market_poll` 中每轮行情更新后调用 `SimTradingService::on_price_update`。
- 限价买入：当最新价 ≤ 委托价或卖一 ≤ 委托价时成交；限价卖出：当最新价 ≥ 委托价或买一 ≥ 委托价时成交。
- 成交价采用 **三价取中**：
  - 买入成交价 = `max(委托价, 卖一价, 最新价)`（取对买方最不利价）
  - 卖出成交价 = `min(委托价, 买一价, 最新价)`（取对卖方最不利价）
- 支持部分成交：当盘口可用量不足时，按对手盘量成交，订单保持 `partially_filled`；无盘口量时回退为一次性全成。
- 滑点配置：按 tick 数在成交价上向不利方向偏移，默认内置规则已设 1–2 tick。

#### 4.1.2 订单类型扩展（已完成）

新增订单类型与状态流转：

| 类型 | 行为 |
|---|---|
| 止损单 / 止损限价 | 触发价达到后转市价/限价平仓。 |
| 止盈单 / 止盈限价 | 触发价达到后转市价/限价平仓；触发方向与止损相反。 |
| 移动止损（Trailing Stop） | 随盈利扩大动态收紧触发价，回撤 `trailing_distance_ticks` 个 tick 触发。 |
| OCO | 止盈与止损同时挂，任一成交后自动撤销另一单；部分成交后也撤销另一单。 |
| 条件单（价格触发） | 当最新价 `>=` 或 `<=` 触发价时生成委托。 |

`sim_orders` 已扩展字段保存触发价、止损/止盈价、OCO 组 ID、父单 ID、条件方向、移动止损距离与参考价。

#### 4.1.3 基础风控（已完成）

- 最大单笔手数限制（账户级 + 品种级）。
- 单品种保证金占用上限 / 集中度上限。
- 账户风险度 ≥ 阈值时 `block_open` 禁止开新仓，≥ 强平线时自动按市价减仓并写入 `sim_risk_events`。
- 下单前检查可用资金、保证金、手续费、风控限制，拒单时返回明确原因。

#### 4.1.4 前端适配（已完成）

- `SimulationPage` 下单面板已增加市价/限价/止损/止损限价/止盈/止盈限价/移动止损/条件单切换。
- 条件单支持 `>=` / `<=` 方向选择；移动止损支持回撤 tick 数输入。
- 委托列表显示类型、价格、触发价、条件方向、已成交/总量与状态。
- 实时行情与费用估算已接入下单面板。

#### 4.1.5 验收（已完成）

- 市价/限价/止损/止盈/移动止损/OCO/双向条件单在 mock 行情下可正常触发、成交、撤销。
- `cargo test --lib` 已新增撮合单元测试：三价取中、部分成交、OCO 互斥、止损/止盈/条件/移动止损触发。
- `cargo test --test sim_trading_test` 已新增集成测试：OCO、条件单、移动止损、部分成交、风控 block_open、强平风险事件。
- Mock E2E 已覆盖条件单页面操作。

### 阶段二：回放训练与绩效分析（P0/P1，约 3–4 周）

目标：把回放页面从占位变为可用训练工具，并提供量化复盘指标。

#### 4.2.1 历史行情回放训练 ✅ 已完成

- 新增 `services/replay_runner.rs`：
  - 选择品种 + 日期范围，从 `klines` 表加载历史 K 线。
  - 按 bar 推进（play/pause/step/speed），隐藏未来数据。
  - 回放过程中行情只推进到当前 bar，禁止查看后续 bar。
- 回放下单：在回放 bar 上调用现有 `place_sim_order`，使用回放当前价撮合。
- 回放状态持久化到 SQLite `sim_replay_sessions` 表，支持暂停后恢复。
- 前端 `MarketReplayPage`：
  - 日期/品种选择器。
  - 播放控制条（播放/暂停/步进/倍速/跳转）。
  - 当前回放 K 线图表（只显示到当前 bar）。
  - 回放中下单面板与持仓/成交列表。
- 新增命令：`start_market_replay` / `stop_market_replay` / `step_market_replay` / `get_replay_state` / `get_replay_klines`。
- 新增 Rust 单元测试：`ReplayRunner` 加载、步进、限价单在回放中成交。

#### 4.2.2 账单导入与动态回放

- 支持导入 CSV/Excel 成交记录（字段：时间、品种、方向、开平、价格、手数）。
- 在 K 线上标记导入的开平仓点，按时间顺序动态回放。
- 每笔导入成交支持添加复盘日志（文字/标签）。
- 与盘立方差异化：不做保证金监控中心自动同步，只做本地文件导入。

#### 4.2.3 绩效分析指标

在 `TradingReviewPage` 增加「绩效统计」Tab，基于 `sim_trades` 计算：

| 指标 | 说明 |
|---|---|
| 总收益率 / 总盈亏 | `(期末权益 - 期初权益) / 期初权益`。 |
| 最大回撤 | 权益曲线从高点到低点的最大跌幅。 |
| 胜率 | 盈利交易次数 / 总交易次数。 |
| 盈亏比 | 平均盈利 / 平均亏损。 |
| 风险回报比 | 单笔平均止盈/止损点数比。 |
| 品种贡献 | 各品种盈亏占比。 |
| 时段贡献 | 按日/周/月/时段统计盈亏。 |
| 持仓时间 | 平均持仓时长、隔夜次数。 |
| 夏普/索提诺（后续） | 需要无风险利率与每日收益。 |

已新增 command `get_sim_performance`，前端用折线图/条形图展示核心指标、品种贡献和时段贡献。

#### 4.2.4 验收

- 回放训练可完整跑完一个交易日，下单、撮合、持仓、资金正确。
- CSV 账单导入后可在 K 线上看到标记。
- 绩效指标与手动计算结果一致。

### 阶段三：图表联动、规则自定义与高级体验（P1/P2，约 3–4 周）

目标：打通分析 → 交易 → 复盘的可视化链路，提升专业用户效率。

#### 4.3.1 K 线交易标记

- 在 `SymbolDetailPage` / `MarketReplayPage` 的 lightweight-charts 上：
  - 标记模拟开平仓点（箭头/圆点）。
  - 绘制持仓成本线、止损线、止盈线。
  - 标记条件单/画线触发线。
- 点击标记可查看订单详情、复盘日记、关联资讯。

#### 4.3.2 合约规则与模拟设置

- 在 `SettingsPage` 已新增「模拟规则」section：
  - 编辑各品种合约乘数、最小变动价位、保证金率、手续费。
  - 配置默认滑点、最大手数、风控阈值。
  - 后续可支持导入/导出规则为 JSON。

#### 4.3.3 数据质量与合规标识

- 模拟盘持仓/成交/资金中显示 `estimated` 状态（如平今/平昨估算、合约规则为默认值）。
- 所有模拟下单按钮、账户卡片、导出报告均显著标注「模拟交易」。
- LLM 复盘报告保留免责声明。

#### 4.3.4 画线下单（可选 P2）

- 在 K 线图上右键/工具栏选择「画线下单」。
- 绘制水平线作为触发价，生成价格条件单。
- 依赖图表交互库能力，可放到阶段三后期评估。

#### 4.3.5 验收

- 设置页可修改规则并即时影响新下单。
- K 线标记与持仓/委托数据一致。
- 模拟标识覆盖所有下单与导出入口。

## 6. 实施建议

1. **先补齐撮合，再做订单类型**：止损/止盈/OCO 都依赖挂单持续撮合能力。
2. **回放训练独立服务**：避免与实时 `market_poll` 耦合，`ReplayRunner` 自行管理时间与行情推进。
3. **风控配置默认保守**：初始最大手数、风险度阈值设置较低，用户可主动放宽。
4. **保持本地优先**：账单、规则、复盘记录优先存 SQLite，导出/导入作为互通手段。
5. **文档同步**：每完成一个阶段，更新 `REQUIREMENTS.md`、`SIMULATION_TRADING_DESIGN.md` 和 `DEVELOPMENT_PLAN.md`。
6. **测试先行**：新增撮合规则必须先写 Rust 单元测试，新增页面操作先写 Mock E2E。

## 7. 验收标准

每个阶段完成后需满足：

1. `cargo test --lib` 与 `cargo clippy` 无错误。
2. `pnpm --dir frontend tsc && lint && test` 无新增错误。
3. Mock E2E 覆盖新增页面路径与关键操作。
4. 涉及真实行情的 client E2E 验证回放与实时模拟不冲突。
5. 所有模拟交易入口保留「模拟」标识，LLM 报告保留免责声明。
6. 文档与代码同步更新。
