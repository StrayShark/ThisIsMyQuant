## 2. 国内期货数据源详解

### 2.1 CTP / SimNow（主源）⭐

- **提供方**：上海期货信息技术公司（上期所子公司）
- **内容**：国内六大期货交易所（上期 SHFE、大商 DCE、郑商 CZCE、中金 CFFEX、能源 INE、广期 GFEX）的期货/期权实时 Tick 行情与交易
- **接入**：参考 vnpy `vnpy_ctp` 网关，封装 CTP API（C++ → Python）
- **成本**：SimNow 仿真免费；实盘需期货公司开户，行情一般免费
- **优点**：官方权威、Tick 级实时、覆盖全市场
- **缺点**：仅当日 Tick，无历史；API 是 C++，需封装；需处理断线重连、前置机切换
- **本项目定位**：**实时行情主源**

> **接入参考**：`vnpy_ctp` 的 `CtpMdApi`（行情）与 `CtpTdApi`（交易）。本项目 v1.0 仅用行情接口，不接交易。

### 2.2 天勤 TQSDK（历史 + 备份）

- **提供方**：信易科技
- **内容**：国内期货实时 + 历史 Tick/K 线、主力连续合约
- **接入**：`tqsdk` Python 库，WebSocket 协议
- **成本**：免费版有延迟与频次限制，专业版付费
- **优点**：历史数据完善、主力连续合约自动换月、Python 原生
- **缺点**：免费版限制多；实时性略逊 CTP
- **本项目定位**：历史回填 + CTP 故障时的实时备份

### 2.3 AKShare（免费聚合）⭐

- **提供方**：开源社区（AKFamily）
- **内容**：聚合新浪、东财、同花顺、金十等公开数据，含期货实时行情、历史 K 线、持仓、库存、基差
- **接入**：`akshare` Python 库，HTTP 抓取
- **成本**：完全免费
- **优点**：无 Key、覆盖广、基本面数据丰富（库存/仓单/资金流向）
- **缺点**：有延迟（非 Tick）、稳定性依赖源站、频次不宜过高
- **本项目定位**：历史回填、基本面指标、宏观数据

### 2.4 金十数据（抓包新闻与基本面上下文）

- **提供方**：金十数据网站
- **内容**：经济数据发布日历、实时财经快讯、期货专题卡片、库存/仓单/产业链图表
- **接入**：网页抓包接口，来源页 `https://datas.jin10.com/#/category/52003`
- **成本**：免费
- **合规**：仅作个人研究与信息聚合参考，注明来源；不作为商业转售数据
- **本项目定位**：纳入 LLM prompt 的宏观/产业新闻上下文，不作为行情主源

抓包通道实现约束：

1. 接口放在独立的 `JinshiNewsFeed`/资讯适配器，不混入 `BaseDataFeed` 行情主链路。
2. 默认低频轮询与本地缓存，失败时静默降级，不影响 CTP/AKShare 行情。
3. 新闻入库保留 `source=jin10`、`category_id`、`display_time`、`title`、`desc/list`、`url`，便于溯源。
4. 对 `52041`、`52042` 这类混合内容做关键词二次过滤，避免海外指数工具污染国内期货分析。

#### 财经日历（宏观数据发布日程）

来源页：`https://rili.jin10.com/`（日视图 WebSocket + HTTP 开放接口）

| 用途 | 接口 | 关键参数/头 |
|---|---|---|
| 按日经济数据 | `GET {JINSHI_RILI_API_BASE}/data/getDay` | `date=YYYY-MM-DD`；`x-app-id: sKKYe29sFuJaeOCJ`、`x-version: 0.28` |
| MCP 备用（需 Token） | `POST https://mcp.jin10.com/mcp` | `tools/call` → `list_calendar`；Bearer `JIN10_MCP_TOKEN` |

Rust 实现：`src-tauri/src/adapters/jinshi_calendar.rs`，Tauri Command `list_calendar_events`。

默认查询未来两周、重要性 ★3 及以上事件（CPI/PPI/非农/美联储决议等），并注入 LLM 分析 prompt。

返回字段（兼容多种命名）：`country`、`pub_time`、`name`/`indicator_name`、`star`、`previous`、`consensus`、`actual`、`unit`。

## 3. 海外期货数据源

### 3.1 盈透 IB（主选）

- **内容**：全球期货（CME/ICE/EUREX 等）实时+历史
- **接入**：`ib_insync` 或 vnpy `vnpy_ib`，需 TWS/IB Gateway 运行
- **成本**：账户佣金，行情订阅费另计
- **本项目定位**：海外期货实时主源（v1.0 可选，P2）

### 3.2 Yahoo Finance（免费参考）

- **内容**：海外期货连续合约日 K（如 `CL=F` 原油、`GC=F` 黄金）
- **接入**：`yfinance` 库
- **成本**：免费
- **本项目定位**：海外品种日级参考，做内外盘联动分析

## 4. 本项目数据架构推荐

```
                    ┌─────────────────────────────────────┐
                    │        DataSource Registry          │
                    │   (统一接口 BaseDataFeed)            │
                    └──────────────┬──────────────────────┘
                                   │
        ┌──────────────┬───────────┼───────────┬──────────────┐
        ▼              ▼           ▼           ▼              ▼
   CTP(实时主)    TQSDK(历史)  AKShare(基本面)  金十(消息)   IB/Yahoo(海外)
        │              │           │           │              │
        ▼              ▼           ▼           ▼              ▼
   ┌─────────────────────────────────────────────────────────────┐
   │              DataBus (事件总线 / 内部 pub-sub)               │
   └──────────────┬──────────────────────────┬───────────────────┘
                  ▼                          ▼
          KLineAggregator              AnalysisContext
          (Tick→K线合成)               (为LLM组装上下文)
                  │                          │
                  ▼                          ▼
            Database(SQLite/时序)        LLM Service
                  │
                  ▼
            WebSocket Hub → 前端
```

### 数据流分层
1. **采集层**：各 DataFeed 插件实现统一 `BaseDataFeed` 接口（订阅、退订、查历史）
2. **总线层**：内部事件总线（参考 vnpy EventEngine），Tick/合约/日志事件解耦
3. **加工层**：K 线聚合器、基本面指标计算
4. **存储层**：SQLite（v1）→ QuestDB/TDengine（v2）
5. **消费层**：WebSocket 推前端、LLM 分析、报告归档

## 6. 合规与风险提示

1. **本项目仅用于数据分析与信息呈现，不提供下单交易能力**，规避金融业务合规风险。
2. CTP 实盘连接需用户自有期货账户，凭据本地加密存储，项目不分发任何账户信息。
3. 抓取类数据源（金十、新浪、东财）仅作聚合参考，标注来源，不商业转售。
4. LLM 生成的分析报告**仅供参考，不构成投资建议**，前端需显著声明。
5. 数据落库默认本地，不上传任何第三方服务器（LLM 仅发送行情/指标文本，不发凭据）。
